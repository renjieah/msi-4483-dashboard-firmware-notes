# MSI 4483 Dashboard 副屏固件长期运行 Bug 说明

设备：MSI 4483 Dashboard 小屏，USB HID 设备 `VID=0x0DB0 / PID=0x4BB6`  
固件：`conprog_4BB6_22100300.bin`

## 现象

我这边遇到两个长期运行后才出现的问题：

1. 小屏运行几天后卡在 `loading` 画面，重启也不能恢复，只能重刷固件。
2. 小屏运行一段时间后出现两个界面叠在一起、闪烁、画面混乱，重启后有时仍然这样。

## Bug 原因

这两个问题不像是单纯的硬件坏了，更像是固件里的状态判断和页面切换逻辑有缺陷。

第一个问题是卡 `loading`。

固件主循环里有一段判断，会拿当前保存的内容版本号去和 `21010100` 比较。正常来说，这应该是某种恢复/更新判断，但实际结果是：只要这个值刚好变成 `21010100`，固件就会主动跳进 loading 模式，而且后面没有正常退出路径。

麻烦点在于，MSI Center 或更新工具可能会往小屏 NAND 里写入带这个版本号的内容包。一旦写入后被固件读到，小屏就会进入 loading 循环。因为这个状态已经落到 NAND/配置区里，所以简单断电重启没用，开机后还是会读到同样的状态，只能重刷恢复。

第二个问题是界面叠加、闪烁。

这个小屏固件里有多个显示页面 slot，可以理解成几套不同的页面状态。正常情况下，slot 8 和 slot 9 应该互斥：开一个，就要先关另一个。但我分析到的代码里，有两处在打开 slot 8 前，没有先把 slot 9 关掉。

这样在某些更新或旋转状态变化后，就可能出现：

1. slot 9 原本还开着。
2. 固件又把 slot 8 打开。
3. 两个 slot 同时认为自己应该显示。
4. 两套界面逻辑一起写同一个屏幕缓冲区。

最后表现出来就是两个界面叠在一起、来回抢画面、闪烁或花屏。如果这个错误状态又被写回 NAND，重启后还是会恢复到这个坏状态，所以也会出现“重启无效”的情况。

## 当前修复思路

我做了一个小补丁，只改 3 个位置：

1. 把触发 loading 死循环的版本号比较值改成不可能匹配的值。
2. 在打开 slot 8 前，先把 slot 9 关闭。
3. 另一个相同逻辑的位置也做同样处理。

改动很小，不是重写固件。补丁脚本和分析记录我会放到 GitHub，方便别人复查。

## 注意

刷固件有风险，刷错可能导致设备不启动。这个说明主要是给遇到同样问题的人参考，不建议没有恢复手段的人直接刷。

## One more thing

除了修 bug，我还顺手做了一个魔改方向：让这个小屏不只显示原厂那几页，而是可以由电脑程序实时推送自定义画面。简单说，就是把它从“只能显示 MSI 固定页面的小屏”，变成一个可以自己设计界面的 480x800 副屏。

目前已经有两部分：

1. 魔改固件：在原固件基础上加了自定义 USB HID 数据通道，可以把画面数据直接写进小屏显示缓冲区。
2. 配套程序：Windows 上的 `PanelRuntime`，用 .NET 8 + WebView2 做界面，可以读取 AIDA64/系统信息，然后把 480x800 的网页画面推到小屏上。

现在这个程序大概能做这些事：

1. 显示自定义仪表盘页面，不再局限于原厂页面。
2. 用网页写界面，HTML/CSS/JS 怎么画，屏幕上就怎么显示。
3. 读取 AIDA64 传感器数据，比如 CPU/GPU 温度、占用率、风扇、功耗等。
4. 读取 Windows 网络和硬盘状态，做成网络页、存储页、温度页等。
5. 支持点击小屏切换页面，比如首页、CPU、散热、网络、硬盘详情。
6. 支持低帧率常驻显示，平时 1fps，交互时提高到 5fps，减少设备压力。
7. 不走原厂 MSI Center 的页面系统，而是电脑实时渲染后推送画面。

目前我做出来的页面更像一个自定义硬件监控面板：中间是主要状态，下面/侧边可以切不同详情页。因为底层是 WebView2，所以后面想改主题、改布局、加图表，主要就是改网页。

这个地方现在比以前好玩的点是：有 AI 之后，改网页门槛低了很多。页面本身就是普通的 HTML/CSS/JS，不是封死在固件里的东西；就算程序打包好了，网页资源也是暴露在外面的，想换布局、换颜色、加一页、删一块信息，都可以直接改网页文件。不会写前端的人，也可以把需求丢给 AI 改一版，再自己试效果。

当前比较稳定的基线是 v72 固件，5fps 连续运行已经做过 8 小时测试。v73 是后续候选版本，还没有完整实机验证。也就是说，它已经不是“只能亮一下”的 demo，但仍然属于折腾项目，不是官方稳定功能。

这个部分更偏折腾和研究，不是普通用户必需；我会把源码、Web 页面、补丁脚本和说明放到 GitHub，方便有兴趣的人自己看。
